  <behavior>

    - Test: init_db creates indexes on videos(channel_id), videos(views DESC), videos(upload_date DESC)
    - Test: upsert_channel inserts a new channel record
    - Test: upsert_channel updates an existing channel (same youtube_id) without creating a duplicate
    - Test: upsert_video inserts a new video record
    - Test: upsert_video updates views, likes, title, description, tags, scraped_at for existing video_id
    - Test: get_videos_by_channel returns all videos for a given channel_id
    - Test: get_channel_stats returns video count, last_scraped, latest_upload_date for a channel
    - Test: tags are stored as JSON string and deserialized back to list on read
  </behavior>
  <action>
    1. Create `database.py` with a `Database` class:
       - `__init__(self, path: Path)` -- path to SQLite DB file, creates parent dirs if needed
       - `init_db()` -- CREATE TABLE IF NOT EXISTS for channels and videos (schema from RESEARCH.md), CREATE INDEX IF NOT EXISTS for the 3 indexes
       - `connect() -> sqlite3.Connection` -- returns connection with Row factory, WAL mode
       - `upsert_channel(channel: Channel)` -- INSERT ... ON CONFLICT(youtube_id) DO UPDATE SET name, handle, url, subscribers, scraped_at
       - `upsert_video(video: Video)` -- INSERT ... ON CONFLICT(video_id) DO UPDATE SET views, likes, title, description, tags, scraped_at
       - `upsert_videos(videos: list[Video])` -- batch upsert in a single transaction
       - `get_videos_by_channel(channel_id: str) -> list[Video]` -- query and return Video dataclasses
       - `get_channel_stats(channel_id: str) -> dict` -- returns {video_count, last_scraped, latest_upload}
       - `get_all_channels() -> list[Channel]` -- for status display
       - Tags handling: serialize to JSON string on write (`json.dumps`), deserialize on read (`json.loads`)

    2. Use stdlib sqlite3 only (per research recommendation, no sqlite-utils).

    3. Schema exactly as specified in RESEARCH.md SQL block:
       - channels: youtube_id TEXT PRIMARY KEY, name, handle, url, subscribers, total_views, description, scraped_at
       - videos: video_id TEXT PRIMARY KEY, channel_id TEXT NOT NULL, title, url, views, upload_date, description, duration, tags, likes, scraped_at, FOREIGN KEY

    4. Write tests FIRST (RED), then implement (GREEN). Use tmp_path fixture for test databases.
