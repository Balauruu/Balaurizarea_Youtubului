# T03: Acquire subcommand with download orchestration, manifest bookkeeping, and gap analysis

**Slice:** S02 — Media Acquisition Skill
**Milestone:** M002

## Description

Wire the source modules (T02) into the `acquire` CLI subcommand that executes a search plan, downloads assets, builds manifest.json, and identifies gaps. This task delivers R004 (asset-to-shot matching with gap identification) and starts the R010 gap lifecycle (pending_generation status).

The acquire command reads a search_plan.json (generated by Claude — the [HEURISTIC] part) and executes it deterministically: for each entry, call the specified source's search/download, place files in the correct typed folder, and update manifest.json. After all downloads, compare shotlist.json shot IDs against manifest mapped_shots to produce the gaps list.

## Steps

1. Implement `acquire.py` orchestrator:
   - Read search_plan.json: list of `{source, query, media_type, shot_ids, dest_folder, limit}`
   - For each entry: look up source in SOURCE_REGISTRY, call `search()`, then `download()` top results
   - Build assets list with: filename, folder, source, source_url, description, license, mapped_shots, acquired_by
   - Atomic manifest write: build complete manifest dict, write to temp file, rename to manifest.json
2. Implement gap identification:
   - Read shotlist.json to get all shot IDs and their visual_need + shotlist_type
   - Compare against manifest mapped_shots
   - Shots not covered → gaps with status `pending_generation`
   - Only flag gaps for acquisition-relevant types (archival_photo, archival_video, document_scan) — animation/map gaps are expected (handled by S03/S04)
3. Add `acquire` subcommand to cli.py: takes topic + search_plan path, creates asset directories, runs orchestrator, prints summary
4. Handle incremental acquisition: if manifest.json already exists, merge new assets (don't overwrite existing entries). Deduplicate by source_url.
5. Write `tests/test_media_acquisition/test_acquire.py`: mock SOURCE_REGISTRY with fake sources, verify manifest output shape matches schema, verify gap identification logic (covered shots, uncovered shots, partial coverage), verify incremental merge, verify atomic write.

## Observability Impact

- **New runtime signals:** Per-entry download progress logged to stderr (`[source] OK/FAIL url (size KB)`), per-source searched/downloaded counts in manifest source_summary, gap count + coverage percentage via `status` subcommand
- **Inspection surface:** `python -m media_acquisition status "<topic>"` reads manifest.json and prints asset count, gap count by status, coverage %, and per-source download stats
- **Failure visibility:** Each search_plan entry that fails logs source + URL + error to stderr. Manifest gaps section lists every uncovered shot with its visual_need for quick triage. Schema validation errors from `validate_manifest()` include asset filename or gap shot_id context
- **Redaction:** API keys never appear in logs or manifest — only env var names in error messages
