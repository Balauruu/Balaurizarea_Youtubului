# Pipeline Learnings

Cross-skill patterns discovered during pipeline runs. Updated after completing a pipeline phase or full run — not after every task. Read at session start alongside CLAUDE.md and CONTEXT.md.

**What belongs here:** Patterns that span multiple skills — cascading failures, handoff friction, resource waste that no single skill's `insights.md` would capture.

**What does NOT belong here:** Per-skill behavior (use that skill's `insights.md`), project-specific decisions (use project's `metadata.md`), user preferences (use auto-memory).

**Format:** One-line entries, date-stamped. When 3+ entries point at the same issue, propose a CLAUDE.md or skill update.
