Create the directory `.claude/skills/visual-orchestrator/prompts/` and write `generation.md`.

**Structure (mirror writer prompt pattern):**

1. **Intro** (1 paragraph) -- Role statement: "You parse narrated chapter scripts into structured shot lists for asset acquisition. Your output is a JSON array of shot objects."

2. **Inputs** -- Script.md (chapter structure, narration prose). Note: no other inputs in v1.3 (VISUAL_STYLE_GUIDE.md deferred to SHOT-09).

3. **Schema** -- Show one complete 9-field JSON shot object with inline annotations:
   - `id`: globally sequential S001-S999, never reset per chapter
   - `chapter`: integer, prologue = 0, chapters from `## N.` headings = N
   - `chapter_title`: exact string from heading (or "Prologue" for chapter 0)
   - `narrative_context`: concise paraphrase of what the narrator says during this shot. Never copy narration verbatim. Typically 1-2 sentences
   - `visual_need`: era + geography + subject. Zero cinematographer language. Specific enough for acquisition search queries
   - `building_block`: must match a vocabulary entry exactly
   - `shotlist_type`: one of exactly 6 values (archival_video, archival_photo, text_overlay, animation, map, document_scan)
   - `building_block_variant`: meaningful variant string (Portrait, Interior, Impact Phrase, etc.), never null
   - `text_content`: populated for all text_overlay shots, null for all others
   - `suggested_sources`: array of source strings (URLs, archive names), may be empty array

4. **Building Blocks** -- Two-column table (Block Name | Description). Start with all 18 Duplessis baseline blocks. Then add 5-10 blocks for broader channel coverage (true crime, unsolved cases, modern events, institutional scandals). Claude has discretion on the exact additional blocks per CONTEXT.md. Consider these candidates from research: Crime Scene Photograph, Mugshot/Identification Photograph, Timeline Diagram, Missing Person Card, Social Media Screenshot, Security Footage Frame, Institutional Seal/Logo.

5. **Granularity Rules** -- Primary trigger: narrative beat changes (new era, location, figure, evidence type). NOT paragraph or sentence boundaries. Safety net: no single shot's narrative_context should span more than ~450 words of narration. Soft density guardrails: minimum 2 shots per chapter (hard), typical chapters produce 8-15 shots (soft). Establishing shot rule: each chapter MUST begin with an establishing/orienting shot (Date Card, Location Map, Landscape Establishing Shot, Archival Footage, Glitch Stinger, or other contextual type -- Claude decides based on chapter content). Chapter 0 parsing rule: any content before the first `## 1.` heading is chapter 0 with chapter_title "Prologue".

6. **Type Routing** -- 6 shotlist_type values with explicit assignment rules. Include the type routing table from research (which building blocks map to which types). Critical rule (SHOT-07): abstract narration with no visual record routes to animation, never archival_photo or archival_video.

7. **Anti-Patterns** -- Dedicated section with 4-6 WRONG/RIGHT side-by-side pairs. Use the 6 pairs from research:
   - Cinematographer language in visual_need
   - Era/geography omitted
   - Vague subject description
   - Production terms (B-roll, handheld)
   - Abstract content routed to archival
   - Narration transcribed verbatim into narrative_context

   Additional explicit anti-patterns (not as pairs, but as named rules):
   - No more than 2 consecutive shots using the same building_block
   - No paragraph-boundary shots (splitting at paragraph breaks instead of narrative beats)
   - text_content MUST be populated for all text_overlay shots, null for all others

8. **Worked Example** -- Synthetic 3-4 sentence narration excerpt from a plausible (invented) true crime or unsolved case. NOT from Duplessis. Map to 2-3 shot entries covering at least 2 different shotlist_types. Demonstrate: a text_overlay establishing shot, an archival detail shot, and one animation-type shot.

9. **Output Format** -- JSON array of shot objects. Write to `projects/N. [Title]/shotlist.json`. Shots ordered by chapter, then by narrative sequence within chapter. Sequential IDs across all chapters.

**Tone:** Directive imperative throughout. "MUST", "Never", "Always", "No exceptions." Match the writer prompt's voice exactly.

**Length target:** ~200 lines. Self-contained -- no external references needed at generation time.
