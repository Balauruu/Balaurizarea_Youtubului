  <behavior>
    ## compute_channel_stats(channel, videos) -> dict
    - Given a channel with 5 videos (views: 100, 200, 500, 1000, 5000): returns avg_views=1360, median_views=500, total_videos=5
    - Given 5 videos with upload_dates spanning 40 days: upload_frequency_days = 10.0
    - Given a channel with 0 videos: returns total_videos=0, all other fields 0 or None
    - Given videos where some have None views: filters None views before computing avg/median
    - Given 1 video: upload_frequency_days is None (cannot compute interval)
    - most_recent_upload returns the latest upload_date string

    ## detect_outliers(channel, videos, threshold=2.0) -> list[dict]
    - Given videos with views [100, 200, 500, 1000, 5000] (median=500): returns videos with views >= 1000 (multiplier 2.0 and 10.0)
    - Returns list sorted by multiplier descending (5000 first, then 1000)
    - Each outlier dict has: title, channel, views, multiplier, upload_date
    - Given all None views: returns empty list
    - Given median=0: returns empty list (no division by zero)

    ## format_stats_table(all_stats) -> str
    - Given 2 channel stats dicts: returns ASCII table with header, separator, and 2 data rows
    - Columns: Channel, Videos, Avg Views, Median Views, Upload Freq, Latest Upload
    - Views formatted with commas (e.g., "1,234,567")
    - Upload frequency shown as "Nd" (e.g., "10d") or "n/a" if None

    ## serialize_videos_for_analysis(all_videos_by_channel) -> str
    - Given a dict mapping channel names to video lists: returns structured text with channel headers and video lines
    - Videos sorted by views descending within each channel
    - Each line includes title, views (comma-formatted), upload_date, tags
  </behavior>
