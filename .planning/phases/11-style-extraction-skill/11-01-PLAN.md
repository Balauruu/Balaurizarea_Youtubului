Create `prompts/extraction.md` — the core extraction instructions Claude follows during skill invocation. This is the most critical file: it determines the quality of STYLE_PROFILE.md output.

Structure the prompt in two passes:

**Pass 1: Reconstruction (conditional)**
- Instructions for detecting auto-caption format (signals listed in SKILL.md)
- Reconstruction rules: rejoin broken lines into natural sentences, restore punctuation, strip all bracket tags ([Music], [Applause], [Laughter]), fix obvious proper noun errors where context makes the correct spelling clear, flag uncertain phrases with [unclear]
- Critical constraint: "You are transcribing, not editing. Preserve the narrator's intended phrasing, sentence length variation, and rhythm. Do NOT add conjunctions to merge short sentences, do NOT smooth irregular rhythm, do NOT normalize sentence length."
- Output: save as `[Title]_clean.md` in same directory as original

**Pass 2: Extraction**
- Read the clean script(s) and `context/channel/writting_style_guide.md` (absorb existing 6 rules)
- Read `context/channel/channel.md` (know what NOT to duplicate — identity statements belong there, not in STYLE_PROFILE.md)

**STYLE_PROFILE.md output format — 4 mandatory sections:**

1. `## Universal Voice Rules` (addresses STYLE-01, STYLE-03, STYLE-04)
   - Extract voice and tone rules that are INDEPENDENT of topic type
   - Each rule follows the format: `### Rule N: [Rule Name]` / `**Definition:**` (syntactic/mechanical instruction, NOT adjective descriptions) / `**Do this:**` (verbatim blockquotes from reference) / `**Not this (generic documentary narration):**` (generated counter-examples showing emotional inflation, intensifiers, clickbait framing)
   - Must absorb and expand the 6 existing rules from writting_style_guide.md per the absorption plan in 11-RESEARCH.md:
     - "No clickbait inflation" -> expand to banned vocabulary list + allowed syntax
     - "Script is narration only" -> add examples of host commentary to avoid
     - "Sources are sacred" -> syntax pattern for labeling speculation vs fact
     - "Pacing through silence" -> specific sentence rhythm patterns (short beat after heavy info)
     - "Open endings are valid" -> becomes the Open Ending Template section
     - "Chapters structure the story" -> absorbed into Narrative Arc Templates
   - Include sentence rhythm analysis subsection: short/long alternation patterns, paragraph cadence, emotional beat shifts — all with verbatim examples from the clean script
   - Every rule must be syntactic or mechanical. Test: if a rule reads like an adjective ("neutral", "clinical", "calm"), rewrite it as a do/don't instruction

2. `## Narrative Arc Templates` (addresses STYLE-02, STYLE-04)
   - Extract chapter structure patterns: chapter count range, act progression, pacing rules, chapter-to-chapter connections
   - Every arc template MUST carry an applicability tag: e.g., `[Cult / Group Radicalization]`, `[Conspiracy / Cover-up]`, `[Unsolved Disappearance]`
   - Extract the hook pattern: quote opening -> compressed story overview -> "this is the true story of..." formula. Mark as reusable but flexible
   - Extract chapter title register: evocative naming examples (DO) vs descriptive naming examples (DON'T)
   - "Truth-Seeking Coda" captured as optional/conditional: `[Optional — when widespread misinformation exists]`
   - Explicit note: "These templates are derived from [N] reference script(s). Non-matching topic types should adapt the universal voice rules but create their own arc structure."

3. `## Transition Phrase Library` (addresses STYLE-01, STYLE-03)
   - Extract 10-20 transition phrases VERBATIM from the reference script(s)
   - Categorize by function: Temporal, Causal, Contrast/Revelation, Escalation, Evidential
   - Format: category heading, then bulleted phrases with brief usage note
   - Explicit instruction: "Do NOT include generic connective language (furthermore, notably, in addition). Only phrases that carry the channel's specific voice."

4. `## Open Ending Template` (addresses STYLE-01)
   - When to use: unsolved cases, cases where resolution is ambiguous or contested
   - Three-part structure: present final evidence -> acknowledge unknowns -> leave weight with audience
   - At least one crafted example in channel voice (generated in the channel's extracted style, not verbatim from reference unless a suitable passage exists)
   - Anti-pattern: "Do NOT provide artificial resolution, consolation, or a 'silver lining' ending"

**Optional sections (Claude's discretion based on what the reference script reveals):**
- `## Hook Patterns` — opening formulas
- `## Chapter Naming Register` — evocative vs descriptive titling
- `## Sentence Rhythm Patterns` — detailed rhythm analysis (if not fully covered in Universal Voice Rules)

**Final instruction in the prompt:**
"After drafting STYLE_PROFILE.md, present a summary to the human: list each section heading, count of named rules, count of verbatim examples, count of transition phrases. Wait for approval before writing the file."
